# My Way — 穗稻忠武的專欄

> 一個人用自己的方式,把每天讀到、查到、想到的事,寫成值得留下來的長文。

**網站:https://polarbear0827.github.io/My-Way/**

這是穗稻忠武的個人出版站。不是內容農場,也不是新聞轉貼——每一篇都經過獨立選題、多來源查證、交叉驗證後才寫成,並標明出處。內容橫跨科技、人文、生活與藝術,加上每日的 AI 趨勢觀察與論文速讀,以及一檔每日 Podcast。

## 內容分區

| 分區 | 內容 | 節奏 |
|------|------|------|
| **專欄** | 科技人文、生活日常、藝術長篇——從一個日常物件或現象出發,寫出歷史、身份、文化與制度的縱深。每篇 ≥ 15 個獨立來源、事實逐句查證。 | 每天 3 篇(09:00 / 12:00 / 18:00) |
| **AI Radar** | 每日掃描 GitHub、Trendshift、Hacker News,精選值得關注的 AI 開源工具與趨勢,附編輯評分。 | 每日 |
| **HF 速讀** | HuggingFace Daily Papers 的高密度繁中快讀:今日趨勢、一句話重點、三件事、五篇論文各自的「為什麼重要 / 帶走什麼 / 下一步」。 | 每日 |
| **Podcast** | 〈AI 懶人祕技〉——把當天 AI 落地脈絡講成一集懶人包,可線上收聽或用 Podcast app 訂閱。 | 每日 |

姊妹站:[Daily Briefing 每日早晚報](https://polarbear0827.github.io/Daily-briefing/)。

## 編輯原則

- **不從記憶寫作。** 每一個事實、數字、引言都實際查到來源 URL 才寫。
- **分析與事實分開。** 推論段落明確標注,不混充事實。
- **呈現多方。** 有爭議的地方並陳各方觀點,不選邊站。
- **圖文相符。** 每張圖都與該節主題直接相關,標明來源與授權。

## 技術

純靜態網站,每篇文章為獨立的伺服器端渲染 HTML(全文 + JSON-LD 結構化資料),CSS 內嵌、零外部依賴,由自動化流水線於內容發布時生成並部署。本站為內容的 SEO 主場(canonical),Substack 為次要轉載。
